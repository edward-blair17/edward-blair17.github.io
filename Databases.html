Artifact 3: Databases

Project: Animal Shelter Dashboard (Grazioso Salvare)
Computational Focus: Full-Stack Migration and RESTful API Integration

Artifact Description

The artifact I selected to enhance is the Animal Shelter Dashboard and CRUD Module. Originally developed in Python using the Dash framework, this project utilized a custom Python module to perform CRUD operations on a MongoDB database. For my enhancement, I ported the entire back-end architecture from Python to Node.js using the Express.js framework. This involved rewriting the CRUD logic to use the MongoDB Node.js driver and creating a RESTful API to manage communication between the dashboard and the database.

Justification for Inclusion

This project demonstrates my proficiency in database management and cross-language architecture. Key areas of growth showcased in this artifact include:

  • Cross-Language Proficiency: Demonstrated the ability to translate algorithmic principles from a Python/Dash environment into a modern Node.js/Express runtime.
  • API Development: Created a secure gatekeeper (REST API) that validates requests before they reach the database, rather than allowing the front-end to connect directly to MongoDB.
  • Scalability: Leveraged Node.js's non-blocking I/O loop to handle concurrent data requests more efficiently than the original synchronous Python script.

The Enhancement: Architecture and Security

The primary focus of this enhancement was moving to a multi-tier architecture that follows industry-standard design patterns for security and performance.

1. Secure Configuration Logic

I transitioned from hard-coded connection strings to a modular security model using a 'secure_config.json' file. This acts as a protected repository for database credentials, ensuring they are not exposed within the application logic.

// secure_config.json
{
  "database": {
    "username": "aacuser",
    "password": "SNHU1234",
    "host": "localhost",
    "port": 27017,
    "dbName": "aac",
    "collectionName": "animals"
  }
}

2. Backend API Logic (Node.js/Express)

The following pseudocode illustrates the structure of the new Express server, which handles database connections and serves animal data through secure API endpoints.

# Node.js Express Server Logic
IMPORT Express, MongoDB Driver
 
FUNCTION loadConfig():
    READ "secure_config.json"
    RETURN parsed JSON object
 
ASYNC FUNCTION connectToDatabase():
    config = loadConfig()
    TRY:
        URI = "mongodb://user:pass@host:port"
        client = WAIT FOR MongoClient.connect(URI)
        RETURN client.db("aac").collection("animals")
    CATCH Error:
        PRINT "Database Status: OFFLINE"
 
app.GET("/api/animals", ASYNC (request, response):
    collection = WAIT FOR connectToDatabase()
    IF collection:
        results = WAIT FOR collection.find(query).toArray()
        SEND results as JSON

Growth & Technical Process

This process taught me the importance of environment-specific debugging. While porting the logic was straightforward, I gained a deeper understanding of how different runtime environments interact with database drivers. By moving to Node.js, I was able to implement a non-blocking architecture that allows for more concurrent data requests, mirroring how professional applications operate at scale.

Overcoming Challenges

A significant technical hurdle arose when I attempted to lock down the database for production. I encountered persistent "Authentication Failed" errors. Overcoming this required a deep dive into the system to locate the MongoDB engine files on the Windows drive and manually adjust security settings to enforce password authentication. This experience strengthened my ability to troubleshoot complex, multi-layered systems where the solution lies at the intersection of code and system configuration.

Meeting Professional Standards

This work demonstrates my ability to design and evaluate computing solutions by implementing innovative techniques that deliver industry-standard value. By utilizing a non-blocking I/O loop, I ensured the database interface could handle multiple concurrent requests without thread-locking. Additionally, I showcased a security-first mindset by utilizing a secure, multi-tier authentication strategy. By separating sensitive credentials from the application logic and gatekeeping the data behind a REST API, I have delivered a solution that meets the rigorous reliability and security standards expected in professional software engineering.
