Artifact 2: Software Design & Engineering

Project: Raspberry Pi Thermostat System (Refactored)
Computational Focus: Modular Architecture & Non-blocking Logic

Project Description

This artifact is an embedded climate control system designed to monitor ambient temperatures and manage heating/cooling states via a State Machine. Originally a single procedural script, I enhanced this system into a modular, object-oriented application capable of real-time responsiveness.

Professional Narrative

My enhancement focused on the transition from simple scripting to software engineering. I addressed the "tight coupling" of the original design by encapsulating hardware interactions into specialized classes. Below is a snippet of the HardwareToolbox, which abstracts the sensors and LEDs from the main logic:

class HardwareToolbox:
    """Manages all physical sensors and indicators."""
    def __init__(self):
        self.i2c = board.I2C()
        try:
            self.thSensor = adafruit_ahtx0.AHTx0(self.i2c)
        except Exception as e:
            self.thSensor = None # Defensive Programming

    def read_temperature_f(self):
        """Defensive wrapper with failure fallbacks."""
        try:
            if self.thSensor:
                t_celsius = self.thSensor.temperature
                return ((9 / 5) * t_celsius) + 32
            return self.last_known_temp
        except Exception:
            return self.last_known_temp

By implementing Non-blocking timing, I allowed the system to perform high-frequency tasks (checking buttons) and low-frequency tasks (updating a remote serial server every 30 seconds) simultaneously. This was achieved using delta-time comparisons rather than thread-freezing sleep() commands.

Self-Reflection

This project successfully met Course Outcome 4 by demonstrating professional techniques to deliver real-world value. The introduction of Hysteresis—a calibrated offset to prevent LED flickering at the setpoint—showcases an understanding of how software logic must account for physical hardware limitations. This journey taught me that professional code isn't just about functionality; it's about building modular, maintainable, and resilient systems.
