Artifact 1: Software Design & Engineering

Project: Raspberry Pi Thermostat Prototype [cite: 3]
Computational Focus: Modularization, Object-Oriented Refactoring, and Non-blocking Logic [cite: 12, 27]

Project Description

This artifact is a climate control system originally developed in CS-350 that integrates a Raspberry Pi with a temperature sensor, LCD screen, and LED indicators[cite: 3, 4]. The software manages Off, Heat, and Cool states while transmitting real-time status updates to a remote server via serial communication[cite: 5]. I selected this project because it serves as a critical bridge between physical hardware and complex software logic[cite: 7].

Professional Narrative

My enhancement focused on transitioning a monolithic script—where hardware logic and global variables were tightly coupled—into a modular, class-based "Toolbox" architecture[cite: 9, 13, 14]. This refactoring followed industry best practices by centralizing hardware configurations into a CONFIG dictionary for improved maintainability[cite: 17, 18].

I engineered a HardwareToolbox class to act as an abstraction layer, ensuring the main controller remains independent of specific sensor mechanics[cite: 23, 24]. A key highlight of this class is the defensive sensor wrapper, which prevents system crashes during hardware failures[cite: 10, 25]:

def read_temperature_f(self):
    """Defensive wrapper for sensor reading."""
    try:
        if self.thSensor:
            t_celsius = self.thSensor.temperature
            t_fahrenheit = ((9 / 5) * t_celsius) + 32
            self.last_known_temp = t_fahrenheit
            return t_fahrenheit
        return self.last_known_temp
    except Exception as e:
        # Failsafe: return last known good value if sensor fails
        return self.last_known_temp

To meet professional standards for responsiveness, I overcame the challenge of managing concurrent tasks without the resource contention often found in threading[cite: 34, 35]. By implementing a non-blocking timestamp comparison loop, I ensured the processor remains highly responsive to physical button interruptions while precisely executing periodic data logging and display refreshes[cite: 36, 37].

Self-Reflection

This project demonstrates my ability to employ well-founded computing practices by transitioning to Object-Oriented Programming (OOP)[cite: 41]. Through the separation of concerns, I engineered a resilient system where peripheral failures do not interfere with core thermostat functions[cite: 30, 31, 32]. Ultimately, this refactoring process created an artifact that is not only functional but also accessible to other engineers for future maintenance or scaling, meeting the rigorous standards expected in the software engineering field[cite: 40, 42].
