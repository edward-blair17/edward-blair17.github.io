Artifact 3: Algorithms & Data Structures

Project: Data-Aware Thermostat System
Computational Focus: Sensor Smoothing & Preference Tracking

Project Description

This artifact is an evolution of a state-machine-driven thermostat originally developed for a Raspberry Pi environment. While the original version relied on reactive "if-then" logic, this enhanced version utilizes sophisticated data structures to ensure data integrity and system stability.

Professional Narrative

In this enhancement, I addressed the challenge of "sensor noise"—tiny, random fluctuations in hardware data that cause system jitter. I replaced raw sensor polling with a sliding window buffer utilizing the deque data structure from the Python collections library. This moving average filter ensures the thermostat reacts to genuine environmental changes rather than hardware glitches.

def get_stabilized_temp(self):
    raw_temp = self.getFahrenheit()
    self.temp_history.append(raw_temp)
    
    avg_temp = sum(self.temp_history) / len(self.temp_history)
    return avg_temp

I further optimized the system by implementing a Preference Logger. This algorithm tracks user behavior over time to establish "Historical Comfort Boundaries," setting the groundwork for predictive climate control by logging the highest and lowest setpoints requested by the user.

Self-Reflection

By completing these enhancements, I successfully met Program Outcome 3 by designing a solution that manages the trade-offs between memory usage and system responsiveness. This experience taught me that the best algorithms are those that most effectively manage the constraints of the specific hardware environment.
