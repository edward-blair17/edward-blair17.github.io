Artifact 2: Algorithms & Data Structures

Project: Data-Aware Thermostat System
Computational Focus: Sensor Smoothing & Preference Tracking

Artifact Description

The artifact I decided to go with was the Thermostat Prototype, originally created in CS-350 Emerging Systems Architectures and Technologies. The original version of this software functioned as a basic reactive system: it read a single temperature data point and immediately triggered a state change (Heat/Cool) based on that lone value. While this is functional, it was susceptible to "jitter" or sensor noise, where a single slightly off-reading could cause the hardware to rapidly toggle on and off.

Justification for Inclusion

I chose to enhance this artifact to demonstrate my proficiency in Algorithmic Thinking and Data Organization. By shifting the logic from a single-point reactive model to a data-driven model, I showcased:

  • Sensor Smoothing: Implementing a moving average to stabilize environmental data.
  • Predictive Logging: Tracking user behavior through historical setpoint analysis.
  • Efficiency and Trade-offs: Using appropriate data structures to manage memory while improving system reliability.

The Enhancement: Data Smoothing and User Preference Tracking

The core of this enhancement involved replacing simple variables with more complex data structures to create a "smarter" thermostat.

1. Implementing a Rolling Buffer (Moving Average)

To solve the problem of sensor noise, I implemented a Moving Average Algorithm. Instead of reacting to one raw temperature reading, the system now stores the last ten readings in a deque.

Code Highlight: Algorithmic Smoothing

def get_stabilized_temp(self):
    # 1. Get current reading
    raw_temp = self.getFahrenheit()
    
    # 2. Append to deque (automatically removes oldest if > 10)
    self.temp_history.append(raw_temp)
    
    # 3. Calculate and return average
    avg_temp = sum(self.temp_history) / len(self.temp_history)
    return avg_temp

This demonstrates the use of a queue-based data structure to maintain a fixed-size memory of 10 items, ensuring the thermostat reacts to actual environmental trends rather than momentary noise.

2. Preference Tracking Algorithm

I added a secondary algorithm to monitor and log the user's historical comfort range. By tracking the ‘user_goal_high’ and ‘user_goal_low’, the system begins to learn the specific comfort boundaries of the resident.

Code Comparison: Preference Tracking

# Track the high and low goals requested by the user
if self.setPoint > self.user_goal_high:
    self.user_goal_high = self.setPoint
    
if self.setPoint < self.user_goal_low:
    self.user_goal_low = self.setPoint

This logging logic allows the system to communicate the user's historical "Comfort Range" directly on the LCD screen, moving the prototype closer to modern "Smart Thermostat" functionality.

Growth & Technical Process

This enhancement allowed me to move beyond basic programming into algorithmic optimization. I learned that a computing solution is only as good as the data it processes. By using a sliding window for temperature readings, I created a much more stable user experience. This process required a deep understanding of how to iterate through data structures and perform arithmetic operations to return a stabilized result that the hardware can safely act upon.

Overcoming Challenges

One challenge was determining the appropriate size for the data buffer. A buffer that is too small does not sufficiently filter noise, while a buffer that is too large makes the thermostat too slow to react to actual temperature changes. That is why I decided on a 10-reading buffer, which provided a balance between responsiveness and stability.

Meeting Professional Standards

This work shows my ability to design and evaluate computing solutions using algorithmic principles. By implementing a smoothing algorithm, I solved the technical problem of sensor noise while effectively managing design trade-offs. By specifically using a small amount of memory to gain a significantly more reliable and technically sound solution, I demonstrated that I can utilize data organization to deliver professional software that accomplishes industry-specific goals.
